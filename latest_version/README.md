# Тренажёр иврита — сбор данных с Pealim

## Текущая структура

- `verbs_list.csv` и `fetch_verbs_fixed.py` — источник и сборка сырых данных.
- `data/source/verbs_data_fixed.json` — канонический сырой JSON.
- `convert_fixed_json_to_trainer.py` — конвертер в формат тренажёра.
- `data/canonical/trainer_verbs.json` и `data/canonical/trainer_verbs.js` — канонические build-артефакты.
- `vendor/` — shared React, ReactDOM and Babel bundles reused by all entry HTML files.
- `trainer_app.jsx` — общий React/Babel app-модуль, который используют основная страница и PWA.
- `trainer_verbs.json` и `trainer_verbs.js` в корне — совместимые копии для старых HTML.
- `tools/build_release.py` — сборка deployable `dist/`.

Такой расклад нужен, чтобы менять данные и UI отдельно и не держать несколько ручных копий одного и того же слоя.

Эти три файла собирают данные о 161 глаголе с pealim.com:

- **fetch_verbs.py** — Python-скрипт парсера
- **verbs_list.csv** — список глаголов
- **README.md** — этот файл

## Что в результате получится

Файл `verbs_data_fixed.json` (~500 КБ) — содержит:
- Все формы спряжения (прошедшее, настоящее, будущее, повелит.)
- Производные (имя действия, причастия, и др.)
- Никуд (огласовки) — корректный, с Pealim
- Корни, биньяны, переводы, предлоги управления

## Как запустить (пошагово)

### Шаг 1. Скачать все три файла на мак

Скачай `fetch_verbs.py`, `verbs_list.csv` и `README.md` (этот файл) **в одну папку**, например в `~/Downloads/hebrew/`.

### Шаг 2. Открыть терминал

- Нажми `Cmd + Space` (открой Spotlight)
- Набери `Terminal`
- Нажми Enter

Откроется чёрное окно с приглашением вида `имя@MacBook ~ %`.

### Шаг 3. Перейти в папку со скриптом

Введи в терминале:

```bash
cd ~/Downloads/hebrew
```

(Если ты положил файлы в другую папку — замени путь.)

### Шаг 4. Проверить, что Python установлен

```bash
python3 --version
```

Должно вывести что-то вроде `Python 3.9.6` или `Python 3.11.x`. На любом современном маке Python 3 уже есть. Если терминал говорит "command not found" — установи Python с сайта python.org.

### Шаг 5. Запустить скрипт

```bash
python3 fetch_verbs.py
```

Скрипт начнёт выводить прогресс:

```
📖 Прочитано 161 глаголов из verbs_list.csv
💾 Кеш страниц будет в папке pealim_cache/
⏱  Ожидаемое время: ~241 секунд

  [  1/161] לכתוב          ✓  биньян=פעל    форм=25  производных=4
  [  2/161] לקרוא          ✓  биньян=פעל    форм=25  производных=3
  ...
```

Это займёт **3-5 минут** (по 1 секунде на каждый глагол + парсинг).

### Шаг 6. Найти результат

После завершения в папке появится файл `verbs_data.json` — это и есть наш собранный словарь.

Также появится папка `pealim_cache/` — это кеш скачанных HTML-страниц, можно её удалить или оставить (на случай, если захочешь пересобрать с другими параметрами без повторных запросов к серверу).

### Шаг 7. Отправить файл обратно

Перетащи `verbs_data.json` в чат — я подключу его к тренажёру.

## Что делать, если что-то пошло не так

### Скрипт прерывается с ошибкой сети

Запусти снова — он использует кеш и продолжит с того места, где остановился.

### "command not found: python3"

Установи Python с https://python.org/downloads/ — выбери macOS installer.

### Какие-то глаголы дали ошибку

В выводе будет видно (`✗ ошибка: ...`). Скрипт всё равно соберёт остальные. Список ошибок будет в конце `verbs_data.json` в поле `failures`. Можно потом отдельно поправить URL для проблемных глаголов в `verbs_list.csv` и перезапустить.

### Сервер Pealim не отвечает

Бывает. Подожди 5-10 минут и запусти снова — кеш сохранит уже скачанные страницы.

## Технические детали (для любопытных)

- Скрипт использует **только стандартную библиотеку Python** — никаких `pip install` не нужно.
- Между запросами стоит пауза в **1 секунду** — чтобы не нагружать сервер Pealim.
- Все скачанные страницы кешируются на диск, повторный запуск не делает повторных запросов.
- Парсинг использует HTML-теги с id вида `AP-3MS` (Active Past 3rd Masc Sing), которые Pealim использует для каждой формы — это надёжнее, чем парсинг текста.

## Сборка release-версии

Чтобы собрать папку для выкладки:

```bash
python3 tools/build_release.py
```

В результате появится `dist/` с deployable-копией проекта и `dist/manifest.json`.
PWA-страница в `dist/pwa/index.html` собирается из того же canonical HTML,
что и корневой `dist/index.html`. Обе страницы используют общие vendor-бандлы из `vendor/`,
чтобы не дублировать React/Babel внутри каждого entrypoint.
